The dig Program

Send one mining turtle down a straight corridor of any size — lit with torches, its floor patched over caves and lava, its loot streamed into an Ender Chest — and it walks itself back to the mouth when it's done.

Command: dig <length> [width] [height] Turtles: 1 (standalone) Modem: not required Pattern: tunnel.lua

1What dig does

dig is the single-turtle driver for the tunnel pattern. It clears a rectangular corridor — length forward, width to the right, height up — and does four jobs along the way:

Cuts the corridor

Clears the whole box level by level in a tidy back-and-forth sweep, so nothing is missed and the turtle never wanders.

Lights it

Drops a torch on the floor every few blocks (from a torch slot) so the corridor stays mob-free. Needs a height of 2 or more.

Patches the floor

Any gap under the corridor — a cave opening, a lava pool — gets a filler block dropped into it so you can walk the tunnel safely.

Dumps & returns

Streams loot into an Ender Chest when full (no trip home), then walks back to the mouth facing its original direction when finished.

Standalone — no fleet needed

dig is one turtle doing one job; it doesn't use rednet or the control station at all. (The tunnel pattern isn't wired into the fleet yet — that's a later step. For multi-turtle box clears, use mine / the control station.)

2What you need

ItemWhy
1× Mining TurtleAny Mining Turtle. An Advanced one gives coloured output. No modem required.
FuelCoal / charcoal in a slot, then refuel all. It needs roughly width × length × height plus the return trip — dig warns and tops up automatically if it can.
Torches (optional)A stack in slot 15 to light the corridor as it goes.
Filler blocks (optional)Cobblestone (or any cheap block) in slot 14 to bridge gaps in the floor.
Ender Chest (optional)One in slot 16 to auto-dump loot; pair it with a chest at base.

Everything optional only engages if it's actually stocked — an empty slot just means that feature is skipped, and dig tells you which ones are on before it starts.

3Get the code in-game

The turtle needs the project files. Bootstrap once with update.lua, then re-run update after any change.

  1. On a fresh turtle, pull the updater once: 
    > wget https://raw.githubusercontent.com/cpopson/ATM10-miningTurtles/master/update.lua update.lua
> update
  2. After every push to GitHub, just run update again.

4Geometry — place it right

The corridor is measured relative to how the turtle faces when you start it. The turtle stands in the mouth — its own block is the first cell — and the tunnel grows:

Length (1st number)Forward — the way the turtle is facing
Width (2nd number)To the turtle's right
Height (3rd number)Straight up

Defaults

Give just a length and you get a 1 wide × 2 tall corridor — the classic walk-through tunnel. Add a width and/or height for bigger bores.

Slot 16 — Ender Chest. Auto-dumps loot in place when the loot slots fill.
Slot 15 — Torches. One on the floor every few blocks (height ≥ 2).

Slots 1–13 — Loot. Mined blocks collect here.
Torches need headroom

A torch is placed from one block up, so a 1-tall tunnel (height 1) skips torches entirely — there's nowhere to stand to place them. Use height 2+ if you want it lit.

6Run it

  1. Place the turtle in the tunnel mouth, facing the dig direction.
  2. Load fuel and any optional slots (section 5), then refuel all.
  3. Run dig with a length, and optionally a width and height: 
    > dig <length> [width] [height]

A typical lit, floor-patched 3×3 bore, 24 long:

> dig 24 3 3
Ender Chest in slot 16 - will auto-dump loot when full.
Torches in slot 15 - placing one every 8 blocks.
Filler in slot 14 - floor gaps will be patched.
Tunnelling 24 long, 3 wide, 3 tall (~216 cells)...
DONE: cleared 215 cells (3 torches, 2 fills, 1 dumps), back at start.
ExampleResult
dig 20A 1-wide, 2-tall walk-through tunnel, 20 long.
dig 32 1 3A 1-wide, 3-tall corridor (extra headroom), 32 long.
dig 24 3 3A 3×3 bore, 24 long.
dig 16 4 2A wide 4×2 hallway, 16 long.

7Reading the output

Before it starts, dig prints one line per optional feature so you can confirm what's on. When it finishes it prints one of two lines:

Success

DONE: cleared 215 cells (3 torches, 2 fills, 1 dumps), back at start.
  • cells — blocks the turtle moved through (one shy of W×L×H; the mouth is the entry, not a dig).
  • torches / fills / dumps — how many torches it placed, floor gaps it patched, and loot dumps it made.
  • back at start — it returned to the mouth block, facing its original heading.

Aborted

ABORTED (no_fuel) after 88 cells, 1 levels.
  stopped at (6,0,-8) h=0 relative to start.

It stops cleanly and tells you where. Common reasons:

ReasonMeaning
no_fuelRan out of fuel mid-dig. Refuel and re-run (it'll re-cut from the mouth).
blocked_unbreakableHit bedrock or another unbreakable block in the corridor.
blocked_retry_exceededSomething kept refilling the space (falling gravel/sand past the retry budget).
dump_place_failedCouldn't place the Ender Chest to dump (obstruction above). Rare.
The stall position

The stopped at (x,y,z) h=… line is relative to the mouth (where you started), so you can find the turtle: x = right, y = up, z = forward (negative), h = its heading.

8Tuning (config.lua)

The reserved slots and torch spacing live in config.lua. Change a value, run update, and it takes effect.

KeyDefaultControls
chestSlot16Slot the Ender Chest lives in.
torchSlot15Slot the torches live in.
fillerSlot14Slot the filler blocks live in.
torchEvery8Blocks between torches along the corridor.
Keep them contiguous

The reserved slots sit at the top of the inventory so the loot region below them stays contiguous. If you move one, keep the trio together (e.g. 16/15/14) so loot never spills into a reserved slot.

9Troubleshooting

SymptomFix
"usage: dig …" and nothing runsYou didn't give a length. The first number is required: dig 20.
"Low fuel…" and it stopsAdd coal/charcoal and refuel all. It needs ~width × length × height plus the trip back.
Corridor is darkNo torches in slot 15, or the tunnel is only 1 tall. Stock slot 15 and use height ≥ 2.
Floor still has holesNo filler in slot 14 — add cobblestone there. (Fill only patches gaps directly under the corridor.)
Loot dropped on the groundNo Ender Chest in slot 16, so the turtle overflowed. Add one (or run shorter tunnels).
Turtle stopped partwayRead the ABORTED (reason) line (section 7). Usually fuel or an unbreakable block.
